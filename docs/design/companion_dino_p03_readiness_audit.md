# MVP-P03 お供恐竜 readiness audit

## 目的

MVP-P01からP02.1までのお供恐竜実装を、本番投入前の観点で棚卸しする。今回は大規模実装や本格アセット生成は行わず、残作業と優先順位を明確にする。

## 現在実装の分類

| 対象 | 分類 | 現状 | 本番前の扱い |
| --- | --- | --- | --- |
| お供恐竜10種類のデータ定義 | 仮実装 | `src/data/companion_dinos.js` に10種類、type、skill、rarity、behavior、asset keyがある | 表示名/説明文の文字化け確認と本番文言化が必須 |
| 所持/選択/強化のsave構造 | 本番投入済みに近い | `companion` 領域、ownedIds、selectedId、levels、eggPending、eggIncubating、hatch時刻、tutorialFlagsを正規化 | main統合時に既存セーブ互換QAを追加 |
| 卵ドロップ | 仮実装 | 1プレイ最大1個、初回補正、取得時save、result表示がある | 確率/演出/取得条件の本番調整が必要 |
| 卵孵化 | 仮実装 | 研究画面の身体強化カテゴリに孵化カードを表示し、DNA/研究Pt/時間経過で完了 | UIの本番化、孵化完了演出、時刻異常時の扱いを追加 |
| ホームのお供パネル | 仮実装 | 中央付近に小パネル、所持/選択/卵状態を表示 | 既存ホームUIとの密度調整、本番アセット適用が必要 |
| お供選択/強化モーダル | 要改善 | 所持リストを表示して選択/強化可能 | 現状5行固定のため10種類を扱うスクロール/ページングが必須 |
| プレイ中追従表示 | 本番投入済みに近い | P02.1で`companionView`がstale cleanupから保護され、プレイヤー右下付近に表示 | 本番sprite/アニメーション、視認補助、負荷管理が必要 |
| 行動AI | 仮実装 | 攻撃、範囲、回復、回収、防御、速度、EXP、ボス特化、雑魚処理、シナジー補助の最小差別化あり | type別効果の検証、数値調整、エフェクト接続が必要 |
| チュートリアル | 仮実装 | 卵取得、研究孵化、お供取得、セット後の通知導線がある | A13系チュートリアルUIへの統合と再表示導線整理が必要 |
| リザルト表示 | 仮実装 | 卵入手時に報酬行を表示 | 取得演出と卵状態の説明を強化 |
| 研究画面孵化カード | 仮実装 | 卵所持/孵化中/完了で状態別カードを表示 | 専用カードアセットと短文整理が必要 |
| debugCompanion導線 | 本番前QA用 | 強制所持/セット/表示、全解放、卵/孵化状態を作れる | main統合時は通常導線に出さない |
| stale cleanupとの整合 | 本番投入済みに近い | `companionView` はactive runtime viewに含まれる | companion projectile/effect追加時に同じ保護が必要 |

## 本番投入済み

- 既存セーブに`companion`がない場合の補完。
- 不正な所持ID、selectedId、levelを正規化する基本処理。
- 卵取得から孵化開始、孵化完了、未所持ランダム取得、全所持時DNA代替報酬までの保存処理。
- ホームでセットしたお供をPlaySceneで取得し、プレイヤー付近に表示する導線。
- P02.1で、stale child cleanupによりお供表示が消える問題を修正済み。

## 仮実装

- 10種類のPNGは存在するが、静止画中心で本番アニメーションではない。
- `effectAssetKey`はデータにあるが、PlaySceneの行動エフェクトとしては十分に使われていない。
- 行動AIは最小差別化で、強さ/頻度/表示フィードバックは仮値。
- ホームのお供モーダルは5行固定で、10種類の所持一覧を本番運用できない。
- 卵取得、孵化、取得後導線は通知中心で、専用演出は未完成。

## 要改善

- `src/data/companion_dinos.js` と既存 companion docs の日本語表示が文字化けしている可能性がある。ユーザー表示に出るため本番前必須。
- お供選択/強化モーダルのスクロールまたはページング。
- 各お供のspriteを小型シート化し、プレイ中の向き/移動に合わせて自然に見せる。
- 10種類の行動エフェクトをPool/上限管理に乗せる。
- debugPerformanceにcompanion関連の数値を追加する。
- コントローラー/仮想マウスでお供モーダルを安定操作できるか確認する。

## 未実装

- お供恐竜10種類の本番アニメーションシート。
- 卵ドロップ専用演出、卵取得通知の本番アセット。
- 孵化完了の演出と取得結果の見せ場。
- お供選択/強化UIの本番アセット。
- 各タイプ専用の攻撃/補助エフェクト実装。
- お供専用のprojectile/effect poolとdebugPerformance計測。
- 本番バランス用のdrop rate、upgrade cost、level効果テーブル。

## 本番前必須

1. 表示文言の文字化け修正。
2. 10種類を扱えるお供一覧UI。
3. 本番最低品質のicon/sprite/effect。
4. PlaySceneでの各タイプ挙動の実効果QA。
5. companion effect/projectileの上限管理。
6. 既存セーブ、卵所持、孵化中、全所持済み、selectedId不正の互換QA。
7. 通常導線での出撃、研究、リザルト、ホームの回帰確認。

## 後回し可

- 4方向完全アニメーション。
- レアリティ別の豪華な孵化演出。
- お供ごとの専用SE。
- 詳細ステータス画面。
- 高度な隊列/複数お供/共進化連携。

## P03結論

P01からP02.1で「卵取得、孵化、所持、セット、プレイ中追従」の縦導線は成立している。ただし、本番投入にはUI表示数、文字化け、本番アセット、AI実効果、負荷管理がまだ不足している。P04以降はアセット、AI、UI、QAを分けて進めるのが安全。
